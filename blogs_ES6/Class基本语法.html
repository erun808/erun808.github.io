    # Class基本语法

    # 1、零碎要点

    ⭐️ 类中定义的方法不加 function 关键字，方法之间不需要逗号分隔，加了逗号会报错。
    ⭐️ 类中的所有方法都是定义在类的 prototype 属性上的。
    ⭐️ 类必须使用 new 关键字调用，否则会报错。
    ⭐️ 类不存在提升，必须保证子类在父类定义之后定义
    ⭐️ 类内部的 this 指向，默认是指向类的实例，但如果单独提取出类中的某个方法，this 的指向会是 undefined，这是因为类内部默认使用严格模式。

        class Logger{
        constructor(){
            this.pname = "pname"
        }
        printName(){
            console.log(this.pname)
        }
    }
    
    const logger = new Logger();
    const {printName} = logger;
    printName() // Error，此时的this指向是undefined

    ⭐️ 扩展：

    • Object.assign() 方法可以合并对象，所以会用来一次性向类中添加多个方法，如下代码所示：
        Object.assign(Person.prototype,{  // 为Person类添加多个方法
        toString(){...},
        toValue(){...}
    })
    • __proto__ 并非JavaScript语言本身特性，是各浏览器提供的，生产环境中使用 Object.getPrototypeOf() 方法获取实例对象的原型
    • ES6为 new 关键字引入了 target 属性，该属性一般用在构造函数中，如果该构造函数是用 new 或 Reflect.constructor() 调用，new.target 会返回包含它的构造函数的引用，否则返回 undefined，所以可以用来判断构造函数是否是用 new 调用的，然后进一步处理。

    # 2、静态方法

    在类中的方法前面加上 static 关键字，就表示该方法不会被实例继承，而只能直接通过类本身来调用，这就是静态方法。如果静态方法中包含 this 关键字，这个 this 指向类而不是实例。（这也说明了静态方法可以和非静态方法重名）。 子类继承父类的静态方法，可用 super 对象调用父类的静态方法：

        class Fa{
        static fn(){
            return "fa"
        }
    }
    class Ch extends Fa{
        static fn(){ // 重写父类静态函数
            return super.fn() // 调用父类静态函数
        }
    }

    super 关键字既可以当作函数使用，也可以当作对象使用。super 作为函数调用时，代表父类的 构造函数，作为对象时，在普通方法中，指向父类的 原型对象；在静态方法中，指向 父类。

    # 3、实例属性

    实例属性除了定义在 constructor() 方法的 this 上，还可以定义在类的最顶层，此时省略 this 关键字。

        class Demo{

        age = 18; // 顶层定义实例属性
        fn = function(){...}; //实例属性可以是方法
        obj = {...}; // 也可以是对象

        constructor(){
            this.name = "tom"; // 常规定义实例属性
        }
    }

    # 4、私有方法和私有属性

    这两种指的是只能在类的内部进行访问的方法和属性，外部不能访问。意思就是在 class 中的 this 可以调用，在 class 外不能用实例或类名来调用。但ES6目前还不提供私有属性和方法的语法，可以用 _ 标记，但实际在外部是可以访问的。
