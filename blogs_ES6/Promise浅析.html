    # Promise浅析

    # 1、定义

    Promise 是异步编程的一种解决方案，比传统的解决方案（回调函数和事件）更合理和更强大。

    Promise本质是一个构造函数，其实例化的对象有且只有3种状态，分别是：

    • pending: 待定状态，也是初始状态
    • fulfilled: 已兑现，代表操作成功（通常也叫做resolved）
    • rejected: 已拒绝，代表操作失败

    Promise实例对象上的状态一经改变就不会再有变化，也就是只能从pending状态变为fulfilled状态或rejected状态，无法从fulfilled或rejected状态变为其他状态。

    # 2、基本用法

    通常使用Promise对异步操作进行处理，在 new Promise() 的时候传入一个函数作为参数，这个函数的参数分别是 resolve 和 reject ，他们是两个函数，由JavaScript引擎提供，不用自己部署，他们的作用就是来改变Promise实例的状态的，resolve 是将Promise的状态从pending转变为fulfilled，在异步操作成功时调用，并且可以将异步操作的结果作为 resolve 的参数传递出去；reject 是将Promise的状态从pending转变为rejected，在异步操作失败是调用，并将异步操作的错误信息作为 reject 的参数传递出去。下面是一个常见的Promise实例创建代码：

        const promise = new Promise((resolve,reject)=>{
        // 这里写异步操作
        if(/*异步操作成功*/){
            resolve(value) // value就是异步操作成功是返回的结果
        }else{
            reject(error) // error就是异步操作失败的信息
        }
    })

    上面代码得到的promise就是Promise的实例，该实例可以使用接下来介绍到的Promise原型上的then方法对异步结果进行操作。

    # 3、Promise.prototype.then()

    Promise实例具有的 then 方法是定义在原型对象上的，它的作用是为Promise实例添加状态改变时的回调函数，then 方法可以接受2个回调函数作为参数（可选），在Promise实例状态改变为fulfilled或rejected的时候调用，他们都接收 new Promise() 的时候传出来的参数，下面是 then 方法的简单例子：

        promise.then( value=>{
        // 对成功结果进行处理
    }, error=>{
        // 对错误结果进行处理
    });

    上面代码中的promise是在基本用法代码中的Promise实例，参数value和error就是resolve和reject函数传入的参数。then 方法中的回调函数在Promise实例状态发生改变的时候才（就）会执行，且 then 方法返回一个新的Promise实例，因此可以使用链式写法：

       promise.then(res=>{
       return res.name
   }).then(res=>{
       console.log(res)  // 此时的res是上一个then方法的返回值 -- res.name
   }) ...

    上面的代码使用 then 方法的链式写法，依次指定了两个回调函数。第一个回调函数完成以后，会将返回结果作为参数，传入第二个回调函数。

    另外Promise原型上还有 catch 方法，Promise.prototype.catch()方法是.then(null, rejection)或.then(undefined, rejection)的别名，用于指定发生错误时的回调函数，也可以链式调用。下面是MDN关于 then 方法链式调用的表示图：

    # 4、Promise.prototype.catch()

    上面提到 catch 方法其实只是 then 方法不定义成功回调函数的特殊写法，最佳实践也是要求不要在 then 方法里面定义 rejected 状态的回调函数（即 then 的第二个参数），而是总使用 catch 方法，理由是使用 catch 方法可以捕获前面 then 方法执行中的错误，也更接近同步的写法 （try/catch） 。下面是 catch 方法的使用：

      promise.then(res=>{
      return res.name
  }).catch(err=>{
      console.log(err)  // 打印捕获的错误（错误可能来自promise或then返回的Promise实例）
  }) ...

    Promise 对象的错误具有“冒泡”性质，上面代码中的 catch 方法不仅可以捕获promise这个实例的错误，还能捕获 then 方法返回的Promise对象的错误。

    # 5、Promise.resolve()

    Promise.resolve() 方法用于将现有对象（非Promise对象）转换为Promise对象，其作用如下面代码所示：

       Promise.resolve('data');
   // 等价于
   new Promise(resolve=>{
       resolve('data')
   })

    Promise.resolve() 的参数有以下几种常见情况：

    1. 参数是一个Promise对象：那么 Promise.resolve() 将不对参数做如何修改，原封不动的返回这个Promise对象。
    2. 参数是基本数据类型：Promise.resolve() 方法会返回一个新的Promise对象，状态为fulfilled，也就是上面代码中表示的情况。
    3. 不带如何参数：同第二种情况一样返回一个状态为fulfilled的Promise对象，只是不带参数。

    # 6、Promise.all()

    Promise.all() 方法用于将多个Promise实例包装成一个新的Promise实例。

       const p = Promise.all([p1,p2,p3,p4]);
   p.then(results=>{
       //p1、p2、p3、p4的状态都为fulfilled才会进入此函数
   }).catch(error=>{
       //p1、p2、p3、p4中有一个的状态为rejected就会进入此函数
   })

    如上是 Promise.all() 的基本使用方法，它接收一个数组参数(也可以是一个Itetator)，数组中的每一项都是Promise对象，如果不是则会调用 Promise.resolve() 方法将其转化为Promise对象。 Promise.all() 返回的实例的状态由其参数的状态所决定，如上面代码中 p 的状态由 p1、p2、p3、p4 共同决定，只有这四个的状态都为 fulfilled ，p 的状态才为 fulfilled，此时 p1、p2、p3、p4 各自的返回值组成一个数组，这个数组作为参数传入 p 的成功回调中。当 p1、p2、p3、p4 中有一个的状态为 rejected ，则 p 的状态为 rejected，此时第一个被 rejected 的实例的返回值作为参数传入 p 的错误回调中。
